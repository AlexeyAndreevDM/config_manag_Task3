# ЭТАП: 1 - Перевод программы в промежуточное представление

# ОБЩЕЕ ОПИСАНИЕ
Разработан ассемблер для учебной виртуальной машины. На данном этапе реализован перевод программы из текстового представления в промежуточное представление. Ассемблер принимает на вход программу в формате JSON и преобразует ее во внутреннее представление в виде списка команд с полями A и B.

# ОПИСАНИЕ ФУНКЦИЙ И НАСТРОЕК
Основной класс Assembler содержит следующие методы:

parse_instruction(op): разбирает отдельную инструкцию из JSON, проверяет корректность кода операции и диапазонов операндов
assemble(input_file, test_mode): основной метод ассемблирования, читает входной файл, парсит инструкции и формирует промежуточное представление
display_internal_representation(): выводит внутреннее представление программы в формате полей и значений, сравнивает с тестовыми примерами
Поддерживаемые команды УВМ:

# Описание языка
Загрузка константы: opcode=22, B=20-битная константа (0-1048575)
Чтение из памяти: opcode=62, B=11-битный адрес (0-2047)
Запись в память: opcode=26, B=11-битный адрес (0-2047)
Умножение: opcode=45, B=11-битный адрес (0-2047)
КОМАНДЫ ДЛЯ ЗАПУСКА
Для запуска ассемблера необходимо выполнить команду:
python assembler.py <входной_файл.json> <выходной_файл.bin> [--test]

# Параметры:

входной_файл.json: файл с программой в формате JSON
выходной_файл.bin: имя выходного бинарного файла (на данном этапе не создается)
--test: необязательный флаг для включения режима тестирования
ПРИМЕРЫ ИСПОЛЬЗОВАНИЯ
Пример входного файла program.json:
[
{"opcode": 22, "B": 722},
{"opcode": 62, "B": 541},
{"opcode": 26, "B": 34},
{"opcode": 45, "B": 94}
]

Запуск в обычном режиме:
python assembler.py program.json program.bin

Запуск в режиме тестирования:
python assembler.py program.json program.bin --test

Ожидаемый вывод в режиме тестирования:
